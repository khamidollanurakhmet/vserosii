Пусть код сам распакует архивы
Представьте, что вы продаете товар на Авито - гитару, чемодан или настольную игру. Покупатель хочет знать, сколько будет стоить доставка, а маркетплейс должен посчитать это автоматически и точно.
Ваша задача - научить модель по данным об объявлении (текст + характеристики + фото) предсказывать четыре величины: вес, высоту, ширину и длину товара. Чем точнее предсказания, тем корректнее стоимость доставки и тем меньше неприятных сюрпризов на пункте выдачи.
Задание:
Автоматический точный расчёт стоимости доставки для продавцов и покупателей Авито подразумевает, что необходимо заранее предсказывать индивидуальные весогабаритные характеристики (ВГХ) товара по его описанию и фото.
Для этого постройте модель, которая для каждого объекта из тестовой выборки предсказывает четыре непрерывные величины:
target_weight - вес (кг),
target_height - высота (см),
target_width - ширина (см),
target_length - длина (см).
Данные:
train.parquet - обучающая выборка,
test.parquet - тестовая выборка,
sample_submission.csv - шаблон отправки,
train.zip и test.zip - изображения товаров (ровно 1 фото на товар).

Тренировочный набор train.parquet с колонками:
item_id -  уникальный ID товара,
order_date - дата оформления заказа,
item_condition - состояние товара,
item_price - цена, ₽,
category_name, subcategory_name, microcat_name - иерархия категорий (ур. 1/2/3),
seller_id, buyer_id - ID продавца/покупателя,
title, description - заголовок и описание объявления,
image_name - имя изображения, соответствует объекту в train.zip.
Таргеты: real_weight (кг), real_height (см), real_width (см), real_length (см).
Тестовая выборка: test.parquet - те же признаки, без таргетов.

Система оценки
Промежуточный балл, отображаемый в таблице результатов до момента завершения раунда, рассчитывается по формуле:

Score = 1 / (1 + LogMAE),
где LogMAE - значение метрики, рассчитанное по указанной выше формуле

Окончательное количество баллов за задание определяется по формуле: 

Points = 200 ⋅ percentileofscore(Scores, Score),
где
Points – количество баллов.
percentileofscore – функция, вычисляющая percentile rank,
Scores – промежуточные баллы, подсчитанные по метрике LogMAE для всех участников,
Score – промежуточный балл, подсчитанный по метрике LogMAE данного участника.
При расчете баллов используются значения метрики участников, чьи решения показали результат лучше, чем приложенный пример ответа (sample_submission.csv) на задачу. Решения с метрикой ниже или равной значению, полученному с помощью примера, оцениваются в 0 баллов.
